Overmind's first-party managed brain

One persistent memory per organization, wired into every agent over MCP. It's what turns a pool of stateless agents into a company that learns — and it's optional by design.

What it does for the org

Memory is Overmind's un-copyable moat: a competitor can ship a launcher, but not your company's accumulated judgment. Wadachi is where that judgment lives.

Recall

Nobody starts cold

Before working, an agent pulls the relevant past — decisions, patterns, prior fixes — with get_context and recall. Context is retrieved, not re-derived.

Store

Learning compounds

What an agent discovers goes back with store_memory and store_decision — rationale and rejected alternatives included. The next run inherits it.

Why

Decisions keep their reasons

Ask why a past choice was made and get the rationale, the context, and what was rejected — so nobody silently re-opens a settled question.

How it's wired

◈

Over MCP, never vendored

Overmind speaks to Wadachi through the Model Context Protocol — a clean process boundary. Overmind handles provisioning and lifecycle; the two stay separate products with a privileged integration.

◈

One brain per organization

Each company gets its own managed brain under the server's data directory. Memory is shared within the org and isolated between them — what one agent writes, its teammates read.

◈

Optional, by contract

Memory is a MemoryProvider contract, not a hard dependency. Point it at Wadachi and the org gains a brain; leave OVERMIND_MEMORY_CMD unset and Overmind runs identically, memoryless. Any MCP memory server works.

Separate project, deliberate integration.  Wadachi is its own open-source repo. Overmind adopts it as the first-party brain (ADR-0004) but never bundles its code — the relationship is MCP + process management, like an editor and a language server.
